### 2.4 Translucent / Jelly Plastic

The inflatable furniture, jelly sandals, and translucent tech aesthetic. This translates to UI panels and overlays.

**Jelly Panel Effect (Canvas 2D):**
```
1. Draw rounded rect with fill color at 65% opacity
2. Add white inner stroke at 30% opacity (simulates plastic edge refraction)
3. Add glossy highlight: top 40% with white gradient (45% to 0% opacity)
4. Underneath: apply a subtle colored shadow (same hue as fill, 20% opacity, 8px blur)
```

### 2.5 Glitter Textures

Glitter should feel like actual light catching on tiny particles, not a static texture overlay.

**Implementation approach (performant):**
Rather than a texture, use a sparse particle field:
- 20-40 tiny white dots (1-2px) scattered across the surface
- Each dot has a random alpha tween: 0.2 to 0.8, duration 400-1200ms (random per dot)
- Dots are static in position but twinkle in brightness
- Creates the "light catching glitter" effect without texture overhead

**Where to use glitter:**
- Item cards tier 5+ background
- Celebration screen backgrounds
- Generator "ready" state aura
- XP bar fill (sparse glitter dots twinkling inside the bar)
- Achievement unlock splash

### 2.6 Bubble / Inflatable Aesthetics

The inflated, puffy, tactile quality of Y2K design. Everything looks squeezable.

**Bubble Button Effect:**
```
1. Base fill: pastel gradient (top lighter, bottom saturated)
2. Highlight: oval white gradient at top-left, 35% opacity, 40% width/20% height
3. Bottom edge: 1px darker line (10% darker than base) for grounding
4. On press: scale to 0.92 (squish), highlight shifts slightly down-right
5. On release: scale to 1.05 then 1.0 (bouncy overshoot, 200ms Back.easeOut)
```

**Inflated Text Effect:**
For celebration text ("AMAZING!", "MAGNIFICENT!"), give letters a puffy quality:
- Thick stroke around text (4-6% of font size) in white
- Fill with pastel gradient
- Slight shadow (2px down, colored to match)
- Scale-in with Bounce.easeOut for the inflated entrance

## 4. Phaser 3 Implementation Guide

### 4.1 Holographic Gradient (Canvas 2D / WebGL)

**Canvas 2D approach (for item card borders):**

```typescript
function createHolographicGradient(
  ctx: CanvasRenderingContext2D,
  x: number, y: number,
  width: number, height: number,
  offset: number // 0 to 1, animated over time
): CanvasGradient {
  const gradient = ctx.createLinearGradient(
    x - width * 0.5, y - height * 0.5,
    x + width * 0.5, y + height * 0.5
  );

  // Shift all stops by offset to create the animated crawl
  const stops = [
    { pos: 0.00, color: '#FF9ED2' }, // pink
    { pos: 0.20, color: '#9DEBFF' }, // blue
    { pos: 0.40, color: '#B8FF9E' }, // green
    { pos: 0.60, color: '#FFD59E' }, // gold
    { pos: 0.80, color: '#D49EFF' }, // lavender
    { pos: 1.00, color: '#FF9ED2' }, // pink (wrap)
  ];

  stops.forEach(stop => {
    const shiftedPos = (stop.pos + offset) % 1.0;
    gradient.addColorStop(shiftedPos, stop.color);
  });

  return gradient;
}

// In your update loop or tween:
// Increment offset by 0.002 per frame for smooth crawl
// Redraw border stroke with new gradient each frame
```

**Phaser 3 WebGL FX approach (for full-card shimmer):**

```typescript
// Requires Phaser 3.60+ with WebGL renderer
const item = this.add.sprite(x, y, 'itemTexture');

// Add shine effect -- simulates holographic light pass
const shine = item.preFX.addShine(0.5, 0.5, 5);
// speed: 0.5 (slow sweep), lineWidth: 0.5, gradient: 5 (rainbow-ish)

// Add glow for the aura effect
const glow = item.preFX.addGlow(0xFFD700, 4, 0, false, 0.1, 16);
// gold color, 4px distance, outer glow
```

**Performance note:** The Canvas 2D gradient approach works in both Canvas and WebGL mode. The FX approach is WebGL-only but much cheaper to render per frame. For m3rg3r, use Canvas 2D gradients for item card rendering (which is already canvas-based via EmojiRenderer.ts) and FX pipeline for post-processing effects on sprites.

### 4.2 Chrome Effect on Canvas 2D

Chrome is a multi-stop linear gradient that simulates curved metallic reflection.

```typescript
function drawChromeRect(
  ctx: CanvasRenderingContext2D,
  x: number, y: number,
  w: number, h: number,
  baseHue: string, // e.g., '#F8A4C8' for pink chrome
  radius: number
): void {
  // Parse base color to derive chrome stops
  // (simplified -- in practice, use a color manipulation utility)

  const gradient = ctx.createLinearGradient(x, y, x, y + h);

  // Pink chrome example stops:
  gradient.addColorStop(0.00, '#F8D0E0'); // top: matte light
  gradient.addColorStop(0.15, '#FFFFFF'); // first highlight band
  gradient.addColorStop(0.30, '#F0A0C0'); // mid pink
  gradient.addColorStop(0.50, '#D87898'); // deepest shadow
  gradient.addColorStop(0.65, '#F8F0F4'); // second highlight
  gradient.addColorStop(0.80, '#F0B8D0'); // recovery
  gradient.addColorStop(1.00, '#F8D0E0'); // bottom: back to matte

  ctx.beginPath();
  ctx.roundRect(x, y, w, h, radius);
  ctx.fillStyle = gradient;
  ctx.fill();

  // Inner edge highlight for depth
  ctx.strokeStyle = 'rgba(255, 255, 255, 0.4)';
  ctx.lineWidth = 1.5;
  ctx.stroke();
}
```

**Use cases in m3rg3r:**
- Shop panel header bar
- Generator tier 3 card frame
- Level number display background
- "Premium" or "Rare" badge backgrounds
- Chrome heart motif on special items

### 4.3 Sparkle / Glitter Particle Effects

**Star Sparkle Particle (ambient background):**

```typescript
// Create a small star texture procedurally
function createStarTexture(scene: Phaser.Scene, size: number = 8): string {
  const key = `star_${size}`;
  const gfx = scene.add.graphics();

  // Draw a 4-point star
  gfx.fillStyle(0xFFFFFF, 1);
  gfx.beginPath();
  const cx = size / 2, cy = size / 2;
  const outer = size / 2, inner = size / 6;
  for (let i = 0; i < 4; i++) {
    const angle = (i * Math.PI) / 2 - Math.PI / 2;
    const nextAngle = angle + Math.PI / 4;
    gfx.lineTo(cx + Math.cos(angle) * outer, cy + Math.sin(angle) * outer);
    gfx.lineTo(cx + Math.cos(nextAngle) * inner, cy + Math.sin(nextAngle) * inner);
  }
  gfx.closePath();
  gfx.fillPath();
  gfx.generateTexture(key, size, size);
  gfx.destroy();
  return key;
}

// Ambient sparkle emitter config
const sparkleConfig: Phaser.Types.GameObjects.Particles.ParticleEmitterConfig = {
  x: { min: 0, max: gameWidth },
  y: { min: 0, max: gameHeight },
  lifespan: { min: 2000, max: 4000 },
  speed: { min: 5, max: 15 },
  angle: { min: 250, max: 290 }, // gentle upward drift
  scale: { start: 0, end: 0, ease: 'sine.inout',
    // Custom: scale up, hold, scale down
  },
  alpha: { start: 0, end: 0 },
  // Use onEmit and onUpdate for sparkle behavior:
  // Frame 0-30%: alpha 0 -> 0.8, scale 0 -> 1
  // Frame 30-70%: alpha 0.8, scale 1 (twinkle with random flutter)
  // Frame 70-100%: alpha 0.8 -> 0, scale 1 -> 0
  frequency: 300, // one new sparkle every 300ms
  quantity: 1,
  blendMode: 'ADD', // key: additive blending makes sparkles glow
  tint: [0xFFFFFF, 0xFFD6E8, 0xD4A5FF, 0xA8D8EA], // white, pink, lilac, blue
};
```

**Merge Burst Particle (on successful merge):**

```typescript
function emitMergeBurst(
  scene: Phaser.Scene,
  x: number, y: number,
  chainColors: { primary: number; secondary: number }
): void {
  const emitter = scene.add.particles(x, y, 'star_8', {
    speed: { min: 80, max: 200 },
    angle: { min: 0, max: 360 },
    scale: { start: 0.8, end: 0 },
    alpha: { start: 1, end: 0 },
    lifespan: 600,
    quantity: 12,
    blendMode: 'ADD',
    tint: [chainColors.primary, chainColors.secondary, 0xFFFFFF],
    rotate: { min: 0, max: 360 },
    gravityY: 50, // slight downward pull for organic feel
    emitting: false,
  });

  emitter.explode(12); // burst, not continuous

  // Auto-cleanup
  scene.time.delayedCall(800, () => emitter.destroy());
}
```

### 4.4 Translucent UI Panels

**Frosted Glass Panel (the "jelly plastic" look):**

```typescript
function drawJellyPanel(
  ctx: CanvasRenderingContext2D,
  x: number, y: number,
  w: number, h: number,
  color: string, // base tint, e.g., '#FF9ED2'
  radius: number
): void {
  // 1. Colored fill at reduced opacity (jelly body)
  ctx.save();
  ctx.globalAlpha = 0.65;
  ctx.fillStyle = color;
  ctx.beginPath();
  ctx.roundRect(x, y, w, h, radius);
  ctx.fill();
  ctx.restore();

  // 2. White inner stroke (plastic edge refraction)
  ctx.save();
  ctx.globalAlpha = 0.3;
  ctx.strokeStyle = '#FFFFFF';
  ctx.lineWidth = 2;
  ctx.beginPath();
  ctx.roundRect(x + 2, y + 2, w - 4, h - 4, radius - 1);
  ctx.stroke();
  ctx.restore();

  // 3. Glossy highlight (top 40%)
  const highlightGradient = ctx.createLinearGradient(x, y, x, y + h * 0.4);
  highlightGradient.addColorStop(0, 'rgba(255, 255, 255, 0.45)');
  highlightGradient.addColorStop(1, 'rgba(255, 255, 255, 0.0)');
  ctx.fillStyle = highlightGradient;
  ctx.beginPath();
  ctx.roundRect(x, y, w, h * 0.4, [radius, radius, 0, 0]);
  ctx.fill();

  // 4. Colored drop shadow beneath (not black)
  ctx.save();
  ctx.shadowColor = color; // same hue
  ctx.shadowBlur = 12;
  ctx.shadowOffsetY = 4;
  ctx.globalAlpha = 0.2;
  ctx.fillStyle = color;
  ctx.beginPath();
  ctx.roundRect(x, y, w, h, radius);
  ctx.fill();
  ctx.restore();
}
```

**Phaser 3 implementation note:** Since Phaser's Graphics object doesn't support rounded rect natively in all versions, the cleanest approach is:
1. Render jelly panels to a canvas texture at boot time (like EmojiRenderer.ts already does)
2. Use the generated texture as a NineSlice or TileSprite for UI panels
3. This avoids per-frame canvas draws for static UI

### 4.5 Holographic Card Border Animation

The signature max-tier visual. Combines the gradient with animation.

```typescript
class HolographicBorder {
  private offset: number = 0;
  private speed: number = 0.003; // color cycle speed

  update(delta: number): void {
    this.offset = (this.offset + this.speed * delta) % 1.0;
  }

  draw(
    ctx: CanvasRenderingContext2D,
    x: number, y: number,
    w: number, h: number,
    borderWidth: number,
    radius: number
  ): void {
    const gradient = ctx.createLinearGradient(
      x + w * this.offset, y,
      x + w * (this.offset + 1), y + h
    );

    // Rainbow stops
    const colors = [
      '#FF6B9D', // pink
      '#FFD93D', // gold
      '#6BCB77', // green
      '#4D96FF', // blue
      '#9B59B6', // purple
      '#FF6B9D', // pink (wrap)
    ];

    colors.forEach((color, i) => {
      gradient.addColorStop(i / (colors.length - 1), color);
    });

    ctx.strokeStyle = gradient;
    ctx.lineWidth = borderWidth;
    ctx.beginPath();
    ctx.roundRect(x, y, w, h, radius);
    ctx.stroke();
  }
}
```

**Performance strategy for animated borders:**
- Only tier 7-8 items get animated borders (max 2-3 on board at once)
- Render the border to a small off-screen canvas
- Update the off-screen canvas every 2nd or 3rd frame (not every frame)
- Apply as a texture to a Phaser sprite overlaid on the item card
- This keeps the main render loop clean while still showing animation

## 6. Sprite Design Guidelines

### 6.1 General Rules for All Y2K Items

- **Style:** Flat with subtle gradients (not fully flat, not 3D). Think "premium sticker" quality.
- **Outline:** 1.5-2px outline in a slightly darker shade of the item's dominant color. Never black outlines.
- **Highlight:** Every item gets a small white/light highlight mark (oval or arc) to imply glossiness.
- **Size consistency:** All items render into the same card frame. Design items to fill ~65% of the card area.
- **Color temperature:** Items should feel warm-to-neutral. Avoid cold, clinical blues.
- **Recognizability:** Items must read clearly at 32x32px (small board view). Silhouette should be distinctive.

### 6.2 Tier Visual Escalation Rules

How items within a chain should visually progress:

| Tier Range | Visual Quality |
|-----------|---------------|
| Tier 1-2 | Simple, matte, single item. Looks hand-drawn/sticker-like. |
| Tier 3-4 | Slight gloss added. Minor detail increase. Colors get slightly more saturated. |
| Tier 5-6 | Visible shine/gloss marks. Added decorative detail (rhinestone dots, small stars). Item starts to "glow." |
| Tier 7 | Premium quality. Multiple shine marks. Complex detail. Item looks "expensive." |
| Tier 8 | Scene/vignette rather than single object. Multiple elements compose the final item. Ambient particle effects make it feel alive. |

### 6.3 Sprite Sheet Recommendations

For efficiency with Phaser 3:
- Render all item sprites as 64x64 or 128x128 PNGs in a texture atlas
- Group by chain (Glamour atlas, Tech atlas, etc.)
- Include "glow" variants for tier 5+ (item + pre-rendered glow halo)
- Include decorative sprites separately: stars, hearts, butterflies, sparkle shapes
- If continuing with emoji-based rendering: the Y2K chains use OBJECTS, not nature -- emoji coverage may be sparse. Custom sprites recommended for Y2K chains.
